◆ IO_Type

enum network_butcher::io::Onnx_model_reconstructor_helpers::IO_Type

Enumerator for the type of the tensor in the Onnx graph.

Enumerator
Input 

Graph input.

ValueInfo 

Either an input of a node or an output of either a node or the graph.

Initializer 

Already initialized value.

Initializer_Input 

Already initialized input of the graph.

Initializer_ValueInfo 

Already initialized io of a node.

Fake_ValueInfo 

Formally an input of a node (but not of the graph), it's not an output of any node. Thus, the model analyzed is not trained and this ValueInfo is not initialized. It should be treated by the Node point of view as a ValueInfo, while by the model point of view as an initializer (thus, it cannot be a tensor to be sent to another device)

NoConnection 

Unknown connection (neither an input nor an output nor a ValueInfo....). It should be impossible.
